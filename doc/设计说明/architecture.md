## 分层架构

### Controller 层

负责处理 HTTP 请求，返回 JSON 响应。

```
com.coolxer.controller/
├── business/
│   ├── asset/        # 资产管理 (API/App/File/Host/IoT/...)
│   ├── operation/    # 运营事件 (ANR/Crash/Network/Performance/...)
│   └── risk/         # 风险管理 (Attack/Baseline/Data/Vulnerability/...)
├── dih/              # AI对话
├── retrieval/        # 数据检索
├── dashboard/        # 仪表盘
└── system/           # 系统管理 (User/Role/Menu/...)
```

### Service 层

业务逻辑处理层，采用接口+实现模式。

```
com.coolxer.service/
├── dih/
│   ├── agent/        # Agent实现
│   │   ├── nl2sql/   # NL2SQL服务
│   │   ├── rag/      # RAG实现
│   │   └── converter/# 数据转换器
│   └── rag/
├── retrieval/        # 数据检索
└── config/           # 配置管理
```

### DAO 层

数据访问层，支持多数据源。

```
com.coolxer.dao/
├── mysql/           # MySQL数据访问
│   ├── entity/      # 实体类
│   └── repository/  # JPA仓库
└── clickhouse/      # ClickHouse数据访问
    ├── entity/
    └── repository/
```

## 多数据源架构

ZenVis 采用双数据源设计，MySQL 和 ClickHouse 分工明确：

### MySQL - 配置中心

- 用户认证和权限
- 角色和菜单配置
- 业务规则配置
- 插件配置
- 对话会话存储

### ClickHouse - 数据中心

- 设备资产数据
- 运营事件数据（ANR/Crash/性能等）
- 风险评估数据
- 用户行为数据
- 时序指标数据

## 配置驱动

ZenVis 核心设计理念是**配置驱动**，通过 JSON 配置文件定义：

- 数据模型（Schema）
- 菜单结构
- 页面配置
- 图表配置
- 业务规则

配置目录：`deploy/open_config/`

      ▼

      ▼
